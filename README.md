# LiveStudio

LiveStudio — сервис для запуска видеотрансляций **без регистрации**. Вставьте ссылку
на поток, задайте время показа и получите современный плеер (в стиле YouTube 2026) с
живым чатом. Плеер и чат можно встроить на любой сайт через iframe или JS-код.

## Возможности

- 🎬 **Современный плеер** с перемоткой, скоростью, картинка-в-картинке и полноэкранным режимом
- ⏰ **Расписание** — обратный отсчёт до начала трансляции
- 💬 **Живой чат** в реальном времени с модерацией (владелец удаляет сообщения / очищает чат)
- 🔌 **Встраивание** через iframe и JS-сниппет
- 🧩 **Источники**: YouTube, Twitch, HLS (.m3u8), MP4/WebM и другие прямые ссылки
- 🔒 **Без регистрации** — настройки владельца хранятся локально в браузере
- 📜 Лендинг, FAQ, документация, Условия использования, Политика конфиденциальности и
  обязательное согласие при первом визите

## Технологии

- [TanStack Start](https://tanstack.com/start) (React 19, SSR) + Vite 7
- Tailwind CSS v4
- Бэкенд: Supabase (PostgreSQL + Realtime) — чат и хранилище каналов
- hls.js для HLS-потоков

## Локальный запуск

```bash
bun install      # или npm install
bun run dev      # запуск дев-сервера
```

Откройте http://localhost:3000

## Переменные окружения

Создайте файл `.env` со следующими значениями (выдаются вашим backend-проектом):

```
VITE_SUPABASE_URL=...
VITE_SUPABASE_PUBLISHABLE_KEY=...
VITE_SUPABASE_PROJECT_ID=...
SUPABASE_URL=...
SUPABASE_PUBLISHABLE_KEY=...
SUPABASE_SERVICE_ROLE_KEY=...   # только для серверной части (удаление сообщений)
```

> `VITE_*` переменные доступны в браузере. `SUPABASE_SERVICE_ROLE_KEY` используется
> только на сервере и никогда не попадает в клиентский бандл.

## Деплой

Проект — полноценное SSR-приложение TanStack Start. Перед деплоем добавьте все
переменные окружения из раздела выше в настройки платформы.

### Vercel

1. Импортируйте репозиторий в Vercel.
2. Build Command: `vite build` · Output: определяется автоматически.
3. Добавьте переменные окружения.
4. Deploy.

### Netlify

1. Подключите репозиторий.
2. Build Command: `vite build`.
3. Добавьте переменные окружения и задеплойте.

### Render

1. Создайте **Web Service** из репозитория.
2. Build Command: `bun install && bun run build`.
3. Start Command: `bun run preview` (или адаптер вашей платформы).
4. Добавьте переменные окружения.

> На некоторых платформах для SSR может потребоваться соответствующий
> nitro/server-пресет. См. документацию TanStack Start по таргетам деплоя.

```
  lib/             helpers + серверные функции чата (chat.functions.ts)
  routes/          index, studio, watch/$channelId, embed/$channelId, faq, docs, terms, privacy

```

## Безопасность

- Чат анонимный и публичный по дизайну.
- Право владельца канала подтверждается секретным `owner_token`, который хранится только
  в браузере владельца; удаление сообщений проверяется на сервере.
