# LiveStudio → Конструктор телеканалов (TV-канал как ТНТ/СТС)

Делаем из текущего проекта полноценный конструктор независимых телеканалов: владелец создаёт канал, настраивает расписание, рекламу, премьеры, логотип и оформление — а все зрители видят **синхронно одно и то же** (по серверному времени), как в эфире.

## Сначала — честно про технические границы

Чтобы не обманывать ожидания:

1. **Синхронный эфир 24/7** — ДА. Мы вычисляем «что играет прямо сейчас» детерминированно по времени (UTC). Все зрители на сайте видят один и тот же момент, как премьера на YouTube. При изменении расписания владельцем — у всех обновляется автоматически.
2. **Реклама каждые N минут, премьеры, анонсы, заглушка «технический перерыв», стиль ТНТ/СТС поверх видео (логотип, плашка «ПРЕМЬЕРА», титры диктора)** — ДА, всё это рисуется нашим веб-плеером поверх потока.
3. **Автонарезка видео на «хайповые отрывки» с голосом диктора** — НЕТ так, как в ТВ. Сервер (Cloudflare Worker) не умеет обрабатывать/перекодировать видео (нет ffmpeg). Реально: владелец сам указывает тайм-коды отрывков и текст анонса, а диктора озвучим браузерным синтезом речи (Web Speech API) либо ссылкой на аудио. Это работает на сайте.
4. **m3u8 / IPTV с тем же эфиром, рекламой и логотипом** — ЧАСТИЧНО. Worker не может «склеивать» видео в один HLS-поток. Мы отдадим динамический endpoint, который редиректит/отдаёт **текущий по расписанию** m3u8-источник (если сам источник — HLS). Наложение логотипа/плашек в IPTV-плеере невозможно без медиасервера. То есть IPTV получит правильный поток по времени, но без графики поверх.
5. Про репозиторий `CrocoUser/files` — изучу формат (реклама в начале потока), и заложу режим «преролл-реклама перед стартом».

Всё остальное из ТЗ — делаем полностью.

## Что строим

### 1. Модель данных (Lovable Cloud)
Расширяем таблицу `channels` одним JSON-полем `config` (вся кастомизация в одном объекте, чтобы не плодить миграции) + поля владения:
- `config jsonb` — расписание, реклама, премьеры, бренд (логотип, цвета, тексты), настройки диктора/анонсов.
- Владение остаётся как сейчас: `owner_token` в БД ↔ localStorage у владельца. Добавим действие «Отказаться от канала» — токен владения у клиента стирается, в БД `owner_token` обнуляется, канал продолжает жить без владельца.
- RLS: разрешим UPDATE по совпадению `owner_token` (через серверную функцию, токен не светим).

```text
config = {
  brand:    { logoUrl, logoPosition, logoSize, accentColor, channelName, font },
  schedule: { timezone, blocks: [ { id, title, type, url, startsAt, durationMin, isPremiere, premiereLabel } ] },
  ads:      { enabled, everyMin, durationMin, items:[{url}], fallbackText, contactText },
  promos:   { items:[{ title, dateText, narratorText, posterUrl, clipUrl, in, out }] },
  filler:   { text, posterUrl }   // «технический перерыв»
}
```

### 2. Движок эфира (ядро, чистая функция времени)
`src/lib/broadcast.ts` — функция `getNowPlaying(config, now)`:
- по расписанию определяет активный блок и смещение внутри него (`seek`);
- вставляет рекламные паузы каждые `everyMin` на `durationMin`;
- определяет режим: `program | ad | promo | filler | premiere | countdown`;
- возвращает что показывать + сколько секунд до следующего перехода (для авто-переключения).
Детерминированность по `now()` = синхрон у всех. Серверное время берём через лёгкий server fn, чтобы не зависеть от часов клиента.

### 3. Плеер-«эфир» (на базе текущего VideoPlayer)
- Авто-переключение источника при смене блока (без перезагрузки страницы).
- Оверлеи в стиле ТНТ/СТС: логотип (позиция/размер/прозрачность по настройкам), плашка **«ПРЕМЬЕРА»** капсом на красном, титры диктора, отсчёт до премьеры в прямом эфире.
- Режимы рекламы (преролл + врезки), заглушка «Трансляция на техническом перерыве» + текст владельца + «заказать рекламу».
- Анонсы: проигрывание отрывка по тайм-кодам + озвучка (Web Speech API) текста диктора.
- **Валидация ссылок**: при сохранении проверяем, что ссылка воспроизводима (YouTube/Twitch/.m3u8/.mp4/iframe-плеер). Если плеера нет — не сохраняем, показываем ошибку. Это же чинит «видео не загружается».

### 4. Студия / редактор канала (`/studio`, `/channel/$id/edit`)
Полная кастомизация во вкладках: Расписание (блоки, премьеры, время), Реклама, Анонсы, Бренд (логотип/цвета/имя/шрифт), Заглушка. Любое сохранение → у всех зрителей применяется автоматически (Realtime-подписка на строку канала).

### 5. IPTV / m3u8 endpoint
`/api/public/channel/$id.m3u8` — отдаёт/редиректит на текущий по расписанию HLS-источник. В UI — кнопка «Ссылка для IPTV». С пометкой про ограничения (без графики поверх).

### 6. Расписание для зрителя
Кнопка в плеере → панель с программой передач (как ЭфирГид), не мешает просмотру.

### 7. Прочее
- Смена/добавление любых ссылок в любой момент.
- Realtime-обновление эфира у всех при правках владельца.

## Технические детали
- Хранение владения: localStorage `livestudio.owned.v1` (как сейчас) + серверная функция `updateChannel` с проверкой `owner_token`.
- Realtime: `ALTER PUBLICATION supabase_realtime ADD TABLE channels`.
- Время: server fn `getServerTime()` + клиентская коррекция дрейфа.
- Озвучка диктора: `window.speechSynthesis` (бесплатно, без ключей).
- Без ffmpeg/перекодирования — всё наложение только в вебе.

## Порядок реализации
1. Миграция: `config jsonb`, RLS на UPDATE, realtime.
2. `broadcast.ts` (движок) + типы конфига.
3. Server fns: `updateChannel`, `getServerTime`.
4. Студия-редактор (вкладки) + валидация ссылок.
5. Плеер-эфир с оверлеями и режимами (реклама/премьера/анонс/заглушка).
6. IPTV endpoint + кнопка.
7. Расписание для зрителя + Realtime-синхрон.
